## 🚀 Quick Start

### Option 1: Docker (Recommended - 1 Minute)
```bash
cp .env.example .env           # Edit with Pinnacle credentials
docker-compose up -d --build   # Start PostgreSQL + app
docker-compose logs -f tracker # Monitor
docker-compose down            # Stop
```

### Option 2: Local Python (5 Minutes)
```bash
python -m venv .venv
source .venv/bin/activate      # or .venv\Scripts\activate (Windows)
pip install -r requirements.txt
cp .env.example .env           # Edit credentials
python -m app.main             # Run
# Ctrl+C to stop
```

### Verify It Works
```bash
pytest tests/ -v    # Run unit tests
# Result: 8/8 passing ✅
```

## 🗂️ Key Features

### Polling & API
- ✅ Independent threads per sport (football, basketball)
- ✅ Cursor-based delta queries (bandwidth efficient)
- ✅ Exponential backoff (1s → 2s → 4s → ... → 60s capped)
- ✅ Respects Retry-After header
- ✅ Infinite retry on transient errors
- ✅ 15-second network timeout

### Database
- ✅ Multi-database support (PostgreSQL, MySQL, SQLite)
- ✅ Bulk inserts via `executemany()` (50-100x faster)
- ✅ Unique constraint deduplication (exact replay prevention)
- ✅ Transaction rollback on error
- ✅ Auto-reconnect (10s retry delay)
- ✅ SQLite auto-schema creation

### Reliability
- ✅ Graceful shutdown (SIGINT/SIGTERM)
- ✅ Signal handlers for clean thread exit
- ✅ No data loss (cursor-based polling + unique constraints)
- ✅ Structured logging (ISO 8601, module-scoped, 4 levels)
- ✅ Exception handling (logged, non-fatal)
- ✅ Fixture refresh every 5 minutes (60 cycles)

### Observability
- ✅ DEBUG: Every cycle, cursor updates
- ✅ INFO: Movements written, fixtures refreshed
- ✅ WARNING: Rate limits, server errors, retries
- ✅ ERROR: DB failures, unrecoverable errors
- ✅ Module-scoped loggers (app.main, app.api_client, etc.)

## 🔧 Architecture Overview

```
┌─────────────────────────┐
│   Pinnacle API (v3)     │
│   endpoints:            │
│   - /odds (delta)       │
│   - /fixtures (metadata)│
└────────────┬────────────┘
             │ HTTPBasicAuth
             │ decimal odds
             │ cursor-based
             ▼
┌─────────────────────────┐
│   PinnacleClient        │
│   - Exponential backoff │
│   - Retry-After header  │
│   - 15s timeout         │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│   parse_odds()          │
│   parse_fixtures()      │
│   (flatten response)    │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│   write_movements()     │
│   - bulk insert         │
│   - executemany()       │
│   - ON CONFLICT IGNORE  │
└────────────┬────────────┘
             │
             ▼
┌─────────────────────────┐
│   PostgreSQL /          │
│   MySQL / SQLite        │
│   (odds_movements)      │
└─────────────────────────┘
```

**Threading:**
- Main thread: Initialization, signal handling, thread lifecycle
- Sport threads (N): Independent polling, separate cursors, auto-reconnect
